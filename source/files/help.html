DicomImageIndexer Help

DicomImageIndexer is a stand-alone program for indexing a collection of DICOM image files, creating a searchable index allowing the images to be found by:
  • File name
  • Transfer Syntax UID
  • Photometric Interpretation
  • Planar Configuration
  • Pixel Representation
  • Pixel Presentation
  • Modality

On a Windows machine, you can run the program by double-clicking it or by launching a command window, navigating to the DicomImageIndexer directory, and typing:

    java -jar DicomImageIndexer.jar

If an out-of-memory error occurs, you may assign more memory to the program by running it in a command window with the command:

    java -Xmx768m -jar DicomImageIndexer.jar

The Index Tab

When the program starts, it looks to see if it has a database. If it does not, it creates the database and populates it by walking the directory tree under the directory specified on the Properties tab. It parses all the files it finds in that tree and creates database entries for all the files that parse as DICOM objects.

The files in the tree can be re-indexed (for example, after changing the directory specified on the Properties tab) by clicking the Rebuild Index button in the footer.

When the index is ready, the Index tab displays a summary of all the unique types of images it found.

Example images from all the image types can be exported to the DICOM Storage SCP whose URL is specified on the Properties tab by clicking the Export All Types button in the footer. The number of example images to export can be specified in the text field to the left of the button.

All the images of a specific type can be exported by clicking the Export button next to the listing of the type.

All the images of a specific type can be listed in the Results tab by clicking the List button next to the listing of the type.

The Search Tab

The Search tab contains a list of all the query fields. The format of the query field imposes these rules:

  • Searches are case-sensitive.
  • A blank field matches a missing element or an element with an empty value.
  • * matches an element, whether it exists or not.
  • *text matches an element value that ends with text.
  • text* matches an element value that starts with text.
  • *text* matches an element value that contains text.
  • text matches an element value that exactly equals text.

When the Search button is clicked, the database is searched for images that match all the query fields, and the Results tab is automatically selected.

The Results Tab

The Results tab lists the results of a query. Each matching file is listed, along with two buttons:

  • The Export button exports the file to the DICOM Storage SCP specified on the Properties tab.
  • The Binary button launches the BinaryDump program and displays the contents of the file in binary form. For DICOM files, the BinaryDump program contains a DICOM parser that displays the binary in colors making it easy to follow the structure. It also contains a menu of functions that show the contents of the file in parsed form. The List Elements menu item can be especially useful because it displays an attached frame that lists all the elements that the parser can parse. Clicking on any element automatically jumps the binary display to the selected element.

The Properties Tab

The Properties tab allows for configuring the image root directory and the DICOM Storage SCP URL.

The location of the BinaryDump program can also be configured, but it can be left blank.
